# /sync — 스프린트 시작 시 실행

새 스프린트를 시작하기 전에 이 커맨드를 실행하세요. 팀원들이 남긴 handoff 기록을 읽고 현재 팀 상태를 파악합니다.

## 실행할 작업

1. CLAUDE.md의 `## 팀 구성`에서 **내 이름과 역할**을 확인한다
   - 이름이나 역할을 찾을 수 없으면 사용자에게 직접 물어본다: "이름과 역할(개발/슬라이드/테스트/오케스트레이션)을 알려주세요"
2. `handoff/` 폴더의 모든 `.md` 파일을 읽는다
3. 각 파일에서 **최신 스프린트 항목**(파일 맨 위, 첫 번째 `---` 이전)만 추출한다
4. 각 항목의 **작성자 이름과 타임스탬프**를 확인해서 누가 언제 마지막으로 기록했는지 표시한다
5. CLAUDE.md의 `## 계약` 섹션을 읽어 현재 계약 상태를 확인한다
6. 아래 형식으로 팀 전체 상태를 요약해서 보여준다

## 출력 형식

```
📋 팀 싱크업 요약 — [현재 시간]
👤 내 정보: [이름] ([역할])

[개발] 스프린트 N (HH:MM) — 작성자: [이름]
  완료: ...
  블로커: ...

[슬라이드] 스프린트 N (HH:MM) — 작성자: [이름]
  완료: ...
  필요한 것: ...

[테스트] 스프린트 N (HH:MM) — 작성자: [이름]
  완료: ...
  스펙 gap: ...

[오케스트레이션] 스프린트 N (HH:MM) — 작성자: [이름]
  전체 진행률: ...
  충돌 지점: ...

⚠️ 계약 변경 사항:
  - [이름]이 변경: (내용)
  - (변경 없으면 "변경 없음")

🎯 [내 이름]([역할])이 이번 스프린트에서 신경 쓸 것:
  - (내 작업에 영향 주는 변경점)
  - (다른 팀원이 나에게 요청한 것 — 요청자 이름 포함)
```

## 주의사항
- `handoff/` 폴더가 없거나 비어있으면: "첫 스프린트입니다. 작업 후 `/handoff` 로 첫 상태를 기록하세요." 안내
- 계약 변경이 하나라도 있으면 ⚠️ 로 강조하고 변경한 사람 이름 표시
- handoff가 없는 역할이 있으면 "⏳ [역할] — 아직 handoff 없음" 으로 표시
- 읽은 후 "싱크 완료 — 작업 시작하세요" 로 마무리

# /handoff — 스프린트 종료 시 실행

스프린트가 끝나면 이 커맨드를 실행하세요. 이번 스프린트에서 한 일을 정리해서 팀원들이 볼 수 있도록 기록합니다.

## 실행할 작업

1. CLAUDE.md의 `## 팀 구성`에서 **내 이름과 역할**을 확인한다
   - 이름이나 역할을 찾을 수 없으면 사용자에게 직접 물어본다: "이름과 역할(개발/슬라이드/테스트/오케스트레이션)을 알려주세요"
   - 확인된 이름과 역할을 이후 모든 기록에 사용한다
2. `date +%H:%M` 으로 현재 시각 타임스탬프를 찍는다
3. 현재 프로젝트 상태를 분석한다 (변경된 파일, 새로 생긴 파일, 에러 여부)
4. 아래 템플릿에 맞춰 handoff 항목을 작성한다
5. `handoff/{역할}.md` 파일의 **맨 위에 삽입**한다 (기존 내용 아래로 밀림)
   - 파일이 없으면 새로 생성
   - 이전 스프린트 기록은 그대로 보존 (삭제하지 않음)
6. 스프린트 번호는 해당 파일의 기존 항목 수 + 1로 자동 계산

## 파일 컨벤션

- 위치: `handoff/` (프로젝트 루트 기준)
- 파일명: `handoff/{역할}.md` (예: `handoff/개발.md`, `handoff/슬라이드.md`)
- 역할당 파일 1개 — 스프린트마다 새 파일 만들지 않음
- 최신 항목이 항상 파일 맨 위 (역순 시간)
- 항목 간 구분자: `---`

## handoff 템플릿

파일 맨 위에 아래 내용을 삽입:

```markdown
## 스프린트 N — HH:MM | 작성자: [이름] ([역할])

### 완료
- (이번 스프린트에서 완성한 것)

### 결정사항
- (Claude와 논의해서 내린 설계 결정. 왜 이렇게 했는지 한 줄)

### 미완 / 블로커
- (못 끝낸 것, 막힌 것. 없으면 "없음")

### 계약 변경
- (함수 시그니처, API 엔드포인트, 데이터 구조 등 다른 역할에 영향 주는 변경)
- (변경 없으면 "없음")

### 다음 스프린트 요청
- (다른 역할에게 필요한 것. 없으면 "없음")

---
```

## 예시: handoff/개발.md (스프린트 3번까지 진행된 상태)

```markdown
## 스프린트 3 — 15:40 | 작성자: 일론 머스크 (개발)

### 완료
- analyzer.py 이상치 탐지 로직 완성
- app.py에 차트 컴포넌트 추가

### 결정사항
- IQR 대신 Z-score 사용 (데이터가 정규분포에 가까워서)

### 미완 / 블로커
- 차트 범례 한글 깨짐 (matplotlib 폰트 이슈)

### 계약 변경
- analyze() 반환값에 z_score 컬럼 추가됨

### 다음 스프린트 요청
- 테스트(아이유): z_score 컬럼 포함한 테스트 케이스 추가 부탁

---

## 스프린트 2 — 15:20 | 작성자: 일론 머스크 (개발)

### 완료
- CSV 파싱 기본 구조 완성
...

---

## 스프린트 1 — 15:00 | 작성자: 일론 머스크 (개발)

### 완료
- 프로젝트 초기 셋업 (/init)
...

---
```

## 주의사항
- 타임스탬프와 작성자 이름은 매 항목에 반드시 포함
- 계약 변경 섹션은 반드시 작성할 것 (변경 없으면 "없음"이라고 명시)
- 결정사항에는 "왜"를 반드시 포함
- 다른 팀원의 handoff 파일은 읽기만 하고 수정하지 않음
- 작성 완료 후 "handoff 기록 완료 — 다음 /sync 때 팀원들이 확인합니다" 로 마무리
